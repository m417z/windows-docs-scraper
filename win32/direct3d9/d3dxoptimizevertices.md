# D3DXOptimizeVertices function

Generates an optimized vertex remapping for a triangle list. This function is commonly used after applying the face remapping generated by [**D3DXOptimizeFaces**](https://learn.microsoft.com/windows/win32/direct3d9/d3dxoptimizefaces).

## Parameters

*pIndices* \[in\]

Type: **[**LPCVOID**](https://learn.microsoft.com/windows/win32/winprog/windows-data-types)**

Pointer to triangle list indices to use for ordering vertices.

*NumFaces* \[in\]

Type: **[**UINT**](https://learn.microsoft.com/windows/win32/winprog/windows-data-types)**

Number of faces in the triangle list.

*NumVertices* \[in\]

Type: **[**UINT**](https://learn.microsoft.com/windows/win32/winprog/windows-data-types)**

Number of vertices referenced by the triangle list.

*Indices32Bit* \[in\]

Type: **[**BOOL**](https://learn.microsoft.com/windows/win32/winprog/windows-data-types)**

Flag indicating index type: **TRUE** if indices are 32-bit (more than 65535 vertices), **FALSE** if indices are 16-bit (65535 or fewer vertices).

*pVertexRemap* \[in, out\]

Type: **[**DWORD**](https://learn.microsoft.com/windows/win32/winprog/windows-data-types)\***

Pointer to a destination buffer that will contain the new index for each vertex. The value stored in *pVertexRemap* for a given element is the source vertex location in the new vertex ordering.

## Return value

Type: **[**HRESULT**](https://msdn.microsoft.com/library/Bb401631(v=MSDN.10).aspx)**

If the function succeeds, the return value is D3D\_OK. If the function fails, the return value can be one of the following: D3DERR\_INVALIDCALL, E\_OUTOFMEMORY.

## Remarks

By default, a mesh uses 16 bit indices when it is created unless the application specifies otherwise. To check whether an existing mesh uses 16-bit or 32-bit indices, call [**ID3DXBaseMesh::GetOptions**](https://learn.microsoft.com/windows/win32/direct3d9/id3dxbasemesh--getoptions) and check for the D3DXMESH\_32BIT flag.
